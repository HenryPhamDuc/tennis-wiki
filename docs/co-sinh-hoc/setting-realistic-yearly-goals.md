Building a custom Python extraction pipeline is the ultimate execution of structural minimalism. By programmatic handling the parsing, you completely remove the friction of manual data entry and ensure your output strictly adheres to your Markdown and YAML standards.

To guarantee the formatting never breaks, we will use **Structured Outputs**. Instead of asking the LLM to format the Markdown directly (which is prone to hallucination), we will force it to return a rigid JSON object containing exactly the fields we need. Then, the Python script will assemble that data into perfect, pristine Markdown.

Here is the extraction script using Python and the `google-genai` SDK.

### 1. Prerequisites

You will need the new GenAI SDK and Pydantic (for structural enforcement). Run this in your PowerShell environment:

### 2. The Extraction Script (`parse_sources.py`)

This script assumes you have a folder of raw text sources (e.g., `.txt` exports of your PDFs or transcripts) and will output formatted `.md` files into a processed directory.

Python
    import os
    import glob
    from google import genai
    from google.genai import types
    from pydantic import BaseModel, Field
    # Initialize client (Requires GEMINI_API_KEY environment variable)
    # Note: If routing to a local model like Ollama, you would swap the client 
    # and use the OpenAI compatible endpoint wrapper.
    client = genai.Client()
    # ---------------------------------------------------------
    # 1. DEFINE THE STRICT DATA STRUCTURE
    # ---------------------------------------------------------
    # Pydantic forces the model to return exactly these fields,
    # preventing it from generating unwanted conversational filler.
    class BiomechanicalData(BaseModel):
        author: str = Field(description="Author or source name, if identifiable. Otherwise 'Unknown'.")
        topic: str = Field(description="The primary biomechanical topic (e.g., Serve, Stretch-Shorten Cycle, Kinetic Chain).")
        extracted_points: list[str] = Field(description="List of extracted technical mechanics as concise bullet points. Preserve mathematical notations exactly (e.g., $F_x$).")
    # ---------------------------------------------------------
    # 2. THE EXTRACTION PIPELINE
    # ---------------------------------------------------------
    def process_source_file(input_path: str, output_dir: str):
        print(f"Processing: {input_path}...")

        with open(input_path, 'r', encoding='utf-8') as f:
            raw_text = f.read()

        # The brutally minimalist prompt
        prompt = f"""    You are a biomechanical data extractor. Analyze the following text.     Extract only the technical tennis mechanics, kinetic chain sequences, and physical performance data.     Exclude all introductory text, metaphors, and narrative filler. Ensure mathematical and vector notations are preserved exactly.    Raw Text:    {raw_text}
        """

        try:
            # Enforce the Pydantic schema
            response = client.models.generate_content(
                model='gemini-2.5-flash',
                contents=prompt,
                config=types.GenerateContentConfig(
                    response_mime_type="application/json",
                    response_schema=BiomechanicalData,
                    temperature=0.0 # Set to 0 for maximum precision and minimal variance
                ),
            )

            data = response.parsed

            # ---------------------------------------------------------
            # 3. ASSEMBLE THE PRISTINE MARKDOWN
            # ---------------------------------------------------------
            # Build the exact Tier 1 YAML frontmatter and Markdown structure
            filename = os.path.basename(input_path).replace('.txt', '.md')
            output_path = os.path.join(output_dir, filename)

            markdown_content = f"""---type: sourceauthor: {data.author}
    topic: {data.topic}
    status: processed---# Extracted Technical Data"""
            for point in data.extracted_points:
                markdown_content += f"* {point}\n"

            with open(output_path, 'w', encoding='utf-8') as f:
                f.write(markdown_content)

            print(f"Successfully generated: {output_path}")

        except Exception as e:
            print(f"Error processing {input_path}: {e}")

    # ---------------------------------------------------------
    # EXECUTION
    # ---------------------------------------------------------
    if __name__ == "__main__":
        INPUT_DIR = "./raw_sources"
        OUTPUT_DIR = "./tier1_notes"

        os.makedirs(INPUT_DIR, exist_ok=True)
        os.makedirs(OUTPUT_DIR, exist_ok=True)

        # Grab all text files in the input directory
        source_files = glob.glob(os.path.join(INPUT_DIR, "*.txt"))

        if not source_files:
            print(f"No .txt files found in {INPUT_DIR}.")

        for file_path in source_files:
            process_source_file(file_path, OUTPUT_DIR)

### Why This Architecture Works

* **Zero Output Hallucination:** Because `temperature=0.0` and we are using `response_schema=BiomechanicalData`, the LLM physically cannot output conversational text like "Here is the data you requested." It can only return the raw data objects.

* **Separation of Concerns:** The LLM handles the semantic heavy lifting (identifying what is biomechanics vs. what is fluff), while Python handles the structural formatting. Python strings are deterministic; they will never accidentally drop a YAML `---` boundary.

* **Preservation of Variables:** The prompt explicitly instructs the model to preserve your variables, ensuring strings like `$F_{xy}$` survive the jump from the raw text into the final markdown bullet points.
